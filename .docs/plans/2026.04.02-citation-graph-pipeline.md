# Citation Graph Pipeline

## High-level Goal

Productionize the citation graph that currently lives as testscript JSON files into a maintained database-backed pipeline. This includes:

1. Storing citation edges in a new `paper_citations` table
2. Adding `pagerank` JSONB columns to `papers` and `authors` tables
3. A new S2 client function to batch-fetch references
4. A new DAG that fetches citation edges, expands the graph one layer out, computes PageRank, and writes scores back to the DB
5. Tests that verify the pipeline end-to-end

## Important Technical Notes

### PageRank computation must use networkx in-process

The graph is ~400k nodes and ~900k edges. NetworkX computes PageRank on this in <10 seconds. No need for distributed computation or external graph databases. The DAG task loads all edges from `paper_citations` into memory, builds the graph, runs PageRank, and bulk-updates the DB.

### External reference expansion is expensive and should be a separate weekly DAG

Fetching references for external nodes (papers not in our DB) requires ~137 S2 API batches of 500. This runs as a separate weekly DAG (`expand_external_references_dag.py`, schedule `'0 8 * * 1'` -- Mondays at 08:00 UTC). Runs 3 hours before the daily DAG (11:00 UTC) so that Monday's PageRank computation includes the newly expanded edges. This is cleaner than a weekday check inside the daily DAG: if it fails on Monday, Airflow retries the weekly DAG independently.

### Idempotency: INSERT ON CONFLICT DO NOTHING

All citation edge inserts use `INSERT ... ON CONFLICT DO NOTHING` on the unique constraint. This makes the DAG safe to retry after partial failures -- duplicate edges are silently skipped.

### Percentile computation

Percentiles are computed across all scored papers (not just our DB papers). The formula: `percentile = bisect_right(sorted_scores, this_score) / total_scored_papers * 100`. This means all papers with the max score get P100, and the percentile represents "what percent of papers have a score <= this". Use `bisect_right` on the sorted score list for O(log n) per paper.

### JSONB column structure

```python
# papers.pagerank
# cited_by_count is IN-GRAPH only (papers in paper_citations that cite this paper).
# This is NOT the total S2 citation count (which lives in s2_metrics.citation_count).
{"score": 0.0000890, "percentile": 99.8, "cited_by_count": 1901, "updated_at": "2026-04-02T10:00:00Z"}

# authors.pagerank
{"median_score": 0.0000054, "percentile": 95.2, "scored_paper_count": 5, "updated_at": "2026-04-02T10:00:00Z"}
```

### Detecting papers needing reference fetching

A paper needs references fetched if it has an `s2_ids->>'s2_paper_id'` but does NOT appear as a `source_s2_id` in `paper_citations`. This avoids refetching papers that returned 0 references (which is valid -- S2 hasn't parsed their bibliography).

To handle this correctly, insert a row with `source_s2_id = paper_id, cited_s2_id = NULL` when a paper returns 0 references. This sentinel marks it as "fetched but empty" so we don't re-query it.

### Unique constraints with NULL sentinel rows

PostgreSQL treats `NULL != NULL`, so the unique constraint on `(source_s2_id, cited_s2_id)` does not enforce uniqueness for sentinel rows. Use two partial unique indexes instead:
- `UNIQUE(source_s2_id, cited_s2_id) WHERE cited_s2_id IS NOT NULL` -- prevents duplicate edges
- `UNIQUE(source_s2_id) WHERE cited_s2_id IS NULL` -- prevents duplicate sentinel rows

### isInfluential flag

The S2 API can return `references.isInfluential` per edge. Store this now even though we only use basic PageRank today -- weighted PageRank will need it later (see issue #104).

### Mapping s2_paper_id to paper_authors

PageRank scores are keyed by `s2_paper_id` (string). The `paper_authors` table uses `paper_id` (FK to `papers.id`). The join path is:
1. Load all `(papers.id, papers.s2_ids->>'s2_paper_id')` pairs into a Python dict
2. Use `paper_authors` to map `paper_id` -> `author_id`
3. Look up each paper's PageRank score by its `s2_paper_id`

This avoids a JSONB index and keeps the query simple. The mapping dict is small (~7k papers).

### Rate limiting

`fetch_paper_references_batch` reuses the existing `_request_with_retry` in `client.py`: burst fast, exponential backoff on 429 (starting at 0.1s, doubling each retry, max 10 retries). The S2 free tier shares a global 1000 req/s pool -- backing off quickly lets other users drain the queue so our next attempt succeeds. No API key needed. This is the same strategy used by all other S2 client functions.

### Timeout for reference batch requests

The `references` field returns the full reference list per paper. Papers with hundreds of references produce large responses. Use `timeout=120` (doubled from the 60s used for metadata batches) to accommodate this.

### Bridging compute_percentiles for author scores

`compute_author_scores` returns `{s2_author_id: {"median_score": float, "scored_paper_count": int}}`. To pass this to `compute_percentiles`, extract the scores first: `{k: v["median_score"] for k, v in author_data.items()}`.

### S2 batch response correlation

The S2 batch endpoint returns results in the same order as the input IDs, with `null` for unknown papers. `fetch_paper_references_batch` uses the input index to correlate: `s2_paper_ids[i]` is the `source_s2_id` for response item `i`. This matches the existing pattern in `fetch_paper_metadata_batch`.

### Error handling

Follow the existing DAG pattern (`enrich_s2_paper_data_dag.py`): try/except per batch, accumulate error counts, print progress, report summary. A single failed batch should not abort the entire run.

### External nodes query performance

`fetch_external_nodes_needing_references` uses a `LEFT JOIN ... WHERE source IS NULL` pattern (not `NOT IN`) to find nodes that appear as `cited_s2_id` but not as `source_s2_id`. This performs well with the indexes on both columns. The query runs once per week on ~900k rows.

### citation_graph_dag.py (worker/dags/citation_graph_dag.py) - NEW

#### Methods

| Method | Arguments | Returns | Description |
|--------|-----------|---------|-------------|
| `fetch_papers_needing_references` | `session: Session` | `List[str]` | Query papers with s2_paper_id that don't yet appear as source in paper_citations. Returns list of s2_paper_ids. |
| `fetch_and_store_references` | `session: Session, s2_paper_ids: List[str]` | `Dict[str, int]` | Batch-fetch references from S2 API (500 per batch), INSERT ON CONFLICT DO NOTHING into paper_citations. Returns counts dict. Papers with 0 refs get a sentinel row (cited_s2_id=NULL). |
| `compute_pagerank` | `edges: List[Tuple[str, str]]` | `Dict[str, float]` | Build networkx DiGraph from edge tuples, run PageRank (alpha=0.85, max_iter=100), return {s2_paper_id: score}. |
| `compute_percentiles` | `scores: Dict[str, float]` | `Dict[str, float]` | Given {id: score}, return {id: percentile}. Uses bisect_right on sorted scores. Semantics: percentile = % of papers with score <= this. |
| `compute_author_scores` | `session: Session, paper_scores: Dict[str, float]` | `Dict[str, Dict]` | Load papers.id -> s2_paper_id mapping, join with paper_authors + authors, compute per-author median PageRank. Returns {s2_author_id: {"median_score": float, "scored_paper_count": int}}. |
| `write_paper_scores` | `session: Session, scores: Dict[str, float], percentiles: Dict[str, float]` | `int` | Bulk-update papers.pagerank JSONB for all papers with an s2_paper_id in the scores dict. Also computes cited_by_count from paper_citations. Returns count updated. |
| `write_author_scores` | `session: Session, author_data: Dict[str, Dict], percentiles: Dict[str, float]` | `int` | Bulk-update authors.pagerank JSONB. Returns count updated. |

#### DAG Structure

```
@dag: citation_graph (schedule: '0 11 * * *', daily at 11:00 UTC, after enrichment DAG)

Task 1: fetch_new_references
  - fetch_papers_needing_references() -> s2_paper_ids
  - fetch_and_store_references(s2_paper_ids)

Task 2: compute_scores
  - Load all edges from paper_citations (excluding sentinel rows)
  - compute_pagerank(edges)
  - compute_percentiles(scores)

Task 3: write_scores
  - write_paper_scores()
  - compute_author_scores()
  - compute_percentiles(author_scores)
  - write_author_scores()

Flow: fetch_new_references >> compute_scores >> write_scores
```
