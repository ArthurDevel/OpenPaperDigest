# User Reading Activity Page

## High-Level Goal

Add a "My activity" page to the user section that surfaces reading activity data already collected by the `user_interactions` table. The page shows aggregate stats (papers seen, read, saved, total reading time) and a chronological reading history with per-paper reading time. This leverages the existing interaction tracking infrastructure (`useInteractionTracker`, `useReadingTracker`, `user_interactions` table) -- the main work is querying/aggregating and displaying.

## Important Technical Notes

### Reading time tracking change

The `useReadingTracker` currently only stores `readingRatio` in metadata. We add `activeTimeSeconds` (integer, seconds) alongside it. This is backward-compatible -- old rows in `user_interactions` simply won't have `activeTimeSeconds` in their JSONB metadata, and the stats query handles `null` gracefully.

### Stats query strategy

The `getInteractionStats` method uses two Supabase queries:

**Query 1 -- Aggregation:** Uses `.select('interaction_type, metadata')` (minimal projection) to fetch interaction rows for the user. Groups client-side by `interaction_type` to get counts. Also sums `metadata.activeTimeSeconds` across all `read` rows (skipping rows where it is `null` -- these are old rows before the tracking enhancement). This ensures `totalReadingTimeSeconds` reflects all-time data, not just the LIMIT 50 detail rows.

**Query 2 -- Reading history detail:** Fetches the 50 most recent `read` interactions, then batch-fetches paper title/authors from `papers` table and slugs from `paper_slugs` table. Generates signed thumbnail URLs via batch `getPaperThumbnailUrls` (plural) from `@/lib/supabase/storage` -- not the singular version, to avoid 50 separate storage requests.

Note: `readAt` maps to the `created_at` column in `user_interactions`.

Note: pagination is intentionally omitted for v1 (capped at 50 most recent reads). Duplicate reads of the same paper are shown as separate rows -- this matches the raw event data and keeps the query simple.

### No migration needed

The `user_interactions` table already exists with a `metadata` JSONB column. Adding `activeTimeSeconds` to metadata requires no schema change.
