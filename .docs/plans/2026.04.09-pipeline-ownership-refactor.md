# Pipeline Ownership Refactor

Created: 2026-04-09
Status: Completed
Branch: `ArthurDevel/dag-architecture-audit`

## Why this exists

The current worker architecture has too many overlapping owners for the same derived state:

- `paper_processing_worker` processes PDFs, writes storage assets, creates slugs, links authors, refreshes author stats, computes `max_author_h_index`, and can fail the whole paper after the PDF work is already done.
- `daily_arxiv_ingest` both enqueues papers and does best-effort author linking plus signal refresh.
- `enrich_s2_paper_data` enriches S2 metadata and also links authors for papers that slipped through earlier flows.
- citation maintenance is split across three DAGs and two incompatible completion models:
  - outbound references use sentinel rows in `paper_citations`
  - inbound citations use `papers.inbound_citations_fetched_at`

This makes failures hard to reason about and creates multiple competing sources of truth.

## Current repo facts

These notes are here so a later session can recover context quickly without re-reading the codebase.

### Processing ownership today

- Queue entrypoint is `papers.client.create_paper()`
- Processing claim loop lives in `worker/dags/paper_processing_worker_dag.py`
- Processing currently calls `_link_authors_for_paper()` after storage/db writes
- `_link_authors_for_paper()` also refreshes author stats and writes `signals.max_author_h_index`
- If S2 author linking fails, the paper is marked `failed` even if PDF processing succeeded

### Ingestion ownership today

- `worker/dags/daily_arxiv_ingest_dag.py` creates queued papers
- The same DAG also batch-links authors and updates `max_author_h_index`
- Other `daily_*_papers_dag.py` source DAGs mostly just enqueue via `create_paper()`

### Enrichment ownership today

- `worker/dags/enrich_s2_paper_data_dag.py` enriches `s2_ids`, `s2_metrics`, `classification`, `publication_info`, `s2_tldr`, `s2_embedding`
- It also links authors for papers with `s2_ids` but no `paper_authors`
- It does not currently own author stats or paper signals end to end

### Citation ownership today

- `worker/dags/citation_graph_dag.py` fetches outbound references for internal papers, computes PageRank, writes `papers.pagerank` and `authors.pagerank`
- `worker/dags/expand_external_references_dag.py` fetches outbound references for one-hop external nodes
- `worker/dags/fetch_inbound_citations_dag.py` fetches inbound citations for internal papers
- `worker/dags/citation_helpers.py` contains shared citation fetch/write helpers
- `paper_citations` currently stores both real edges and outbound sentinel rows
- `papers.inbound_citations_fetched_at` separately tracks inbound completion

## Target architecture

Each stage gets one job and one ownership boundary.

### 1. Source DAGs

Responsibilities:

- discover papers
- normalize source-specific metadata/signals
- enqueue papers with `create_paper()`

Non-responsibilities:

- author linking
- S2 metadata enrichment
- author stats refresh
- citation graph maintenance

### 2. Processing worker

Responsibilities:

- claim queued papers
- download/process PDF
- write storage artifacts
- write paper processing fields/status
- create slug

Non-responsibilities:

- author linking
- author stats
- `max_author_h_index`
- citation graph work

Invariant:

- once PDF/storage processing succeeds, later S2 failures must not flip the paper to `failed`

### 3. Enrichment DAG

Single owner of post-processing metadata:

- `s2_ids`
- `s2_metrics`
- `classification`
- `publication_info`
- `s2_tldr`
- `s2_embedding`
- `authors`
- `paper_authors`
- author stats on `authors`
- `signals.max_author_h_index`

### 4. Citation maintenance DAG

Single owner of citation graph state:

- citation node discovery/progress
- outbound fetch status
- inbound fetch status
- graph expansion boundary
- PageRank recomputation

## Planned schema changes

### Paper progress columns

Add explicit timestamps on `papers`:

- `s2_enriched_at`
- `s2_last_attempted_at`
- `author_links_synced_at`
- `signals_refreshed_at`

Optional diagnostic field:

- `s2_enrichment_error`

### Citation node state

Add `citation_node_state` table so fetch progress is explicit and `paper_citations` can become a pure edge table.

Proposed shape:

- `s2_paper_id` primary key
- `paper_id` nullable FK to `papers.id`
- `is_internal` boolean
- `hop` integer nullable
- `discovered_at`
- `outbound_fetched_at`
- `inbound_fetched_at`
- `last_error`
- `updated_at`

- add explicit paper progress timestamps

- enrichment DAG has tasks for metadata enrichment, author sync, author stats refresh, and signal refresh

### Phase 3: unify citation state and orchestration

- replace mixed sentinel/timestamp tracking with explicit citation node state
- make `paper_citations` a pure edge table
- collapse citation maintenance into one DAG

- `worker/dags/citation_helpers.py`
- `worker/dags/citation_graph_dag.py`
- `worker/dags/expand_external_references_dag.py`
- `worker/dags/fetch_inbound_citations_dag.py`

Success markers:

- fetch progress is queryable from `citation_node_state`
- old sentinel-based selection paths are gone
- one DAG owns citation maintenance

Status: [x]

### Phase 4: validation and cleanup

Goal:

- add/update focused tests
- verify the new ownership boundaries and success criteria
- leave the plan file updated with what shipped

Status: [x]

### Phase 5: align manual backfills to the four ownership domains

Goal:

- add four manual backfill DAGs that mirror the same source / processing / enrichment / citation ownership boundaries
- ensure each domain backfill walks the current missing-state backlog instead of owning unrelated state

Planned file changes:

- `worker/dags/source_backfill_dag.py`
- `worker/dags/processing_backfill_dag.py`
- `worker/dags/enrichment_backfill_dag.py`
- `worker/dags/citation_backfill_dag.py`
- shared scoring/helpers where scheduled and backfill citation flows overlap

Success markers:

- source backfill repairs only source-owned metadata
- processing backfill repairs only processing-owned artifacts
- enrichment backfill repairs only S2/author/signal state
- citation backfill repairs only citation graph state and PageRank

Status: [x]

## Success criteria

- a paper can be processed successfully even if Semantic Scholar is unavailable after the PDF work
- each derived field family has one writer/owner
- citation fetch progress is explicit and queryable without sentinels
- the architecture no longer depends on duplicated author-linking logic across multiple DAGs
- the repo contains enough written context here to resume work without re-discovering the whole design
