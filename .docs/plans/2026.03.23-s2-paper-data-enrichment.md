# Semantic Scholar Paper Data Enrichment

## High-Level Goal

Enrich papers in our database with comprehensive metadata from the Semantic Scholar API. This stores the data needed to build features like citation graphs, paper classification, venue filtering, and open access links in the future. Additionally, start persisting arXiv metadata (categories, DOI, journal_ref) that we currently parse but throw away during ingestion.

The enrichment runs as a separate scheduled DAG (`enrich_s2_paper_data`), decoupled from ingestion. No tracking column needed — `updated_at` gets bumped on enrichment, and `s2_ids IS NOT NULL` identifies already-enriched papers.

**New columns on `papers` table:**

| Column | Type | Contents |
|--------|------|----------|
| `s2_ids` | JSONB | All S2 identifiers (paper_id, corpus_id, doi, pmid, pmcid, dblp, mag, acl) |
| `s2_metrics` | JSONB | Citation counts (citation_count, reference_count, influential_citation_count) |
| `classification` | JSONB | ArXiv categories + S2 fields of study + publication types |
| `publication_info` | JSONB | Venue, journal, dates, open access info |
| `s2_tldr` | TEXT | S2's model-generated one-line summary |
| `s2_embedding` | vector(768) | SPECTER v2 embedding |

**JSONB structures:**

## Important Technical Notes

### S2 batch endpoint field request

The S2 `/paper/batch` endpoint accepts a `fields` query parameter as a comma-separated string. Requesting `embedding.specter_v2` is required to get v2 vectors (default is v1). All fields must be requested in a single string:

```
fields=paperId,corpusId,externalIds,citationCount,referenceCount,influentialCitationCount,s2FieldsOfStudy,publicationTypes,venue,publicationVenue,journal,year,publicationDate,tldr,isOpenAccess,openAccessPdf,embedding.specter_v2
```

Max 500 paper IDs per batch request. Response limited to 10 MB. IDs must be prefixed with `ARXIV:` (e.g. `ARXIV:2301.00001`), matching the existing pattern in `fetch_paper_authors_batch`.

### S2 `tldr` field is an object, not a string

The S2 API returns `tldr` as `{"model": "tldr@v2.0.0", "text": "One-line summary here"}`. The `fetch_paper_metadata_batch` parser must extract `data["tldr"]["text"]` and store only the text string. If `tldr` is present but `text` is missing/empty, treat as no data.

### S2 batch response null handling

The S2 batch endpoint returns `null` for papers it cannot find. The parser must skip null entries in the response array (matching the existing pattern in `fetch_paper_authors_batch` line 199). Papers with no S2 match should have `s2_ids` set to `{}` (empty dict) rather than left `NULL`, so they are excluded from future enrichment queries.

### ArXiv categories extraction

The arXiv ingest DAG (`daily_arxiv_ingest_dag.py`) currently does its own XML parsing in `fetch_new_arxiv_papers()` and only extracts `arxiv_id`, `title`, `authors_str`, `abstract`. The XML entries already contain `<arxiv:doi>`, `<arxiv:journal_ref>`, and `<category term="...">` elements but these are ignored. The change adds extraction of these three fields from the same XML.

After `create_paper()` returns, the classification column is updated via a direct SQLAlchemy query on `PaperRecord` within the same session, since `create_paper()` commits internally and detaches the record:

```python
session.query(PaperRecord).filter(
    PaperRecord.paper_uuid == paper_dto.paper_uuid
).update({"classification": classification_dict})
```

### ArXiv classification is for new papers only

The arXiv ingest DAG only writes `classification` for newly created papers (inside the `create_paper()` success path). Existing papers that were skipped ("already exists") do not get backfilled with arXiv categories in this flow. This is intentional -- the S2 enrichment DAG handles existing papers, and `build_classification_dict` will merge S2 data with whatever `classification` data already exists (or `None`).

### Session commit behavior in arXiv ingest

`create_paper()` commits internally via `create_paper_record()`. The subsequent `session.query(...).update({"classification": ...})` is uncommitted until the outer `database_session()` context manager exits. If a later paper in the loop raises an exception that triggers rollback, classification updates for previously successful papers in the same session will be lost. This is acceptable -- classification is best-effort and the S2 enrichment DAG will fill in gaps.

### Existing `to_orm()` gap

The existing `Paper.to_orm()` does not include `abstract` or `embedding`. The new fields follow this same pattern: `s2_embedding` is ORM-only (not on the DTO). The 5 JSONB/text fields are added to both DTO and `to_orm()` because they are useful in business logic, unlike raw embeddings.

### Classification merge strategy

When the enrichment DAG enriches a paper that already has `classification` data (from arXiv ingest), `build_classification_dict` must:
1. Start from `existing_classification or {}`
2. Preserve all existing `arxiv_*` keys untouched
3. Add/overwrite `s2_*` keys (`s2_fields_of_study`, `s2_publication_types`)

### JSONB column type

New JSONB columns should use `Column(JSONB)` from `sqlalchemy.dialects.postgresql` rather than `Column(JSON)`. JSONB supports GIN indexing and containment queries. Note: the existing `signals` and `summaries` columns use `Column(JSON)` -- migrating those is out of scope.

### Rate limits

With S2 API key: batch endpoint is 10 req/s. One batch call handles 500 papers. The enrichment DAG processes 500 papers per loop iteration with a single API call per iteration, well within limits.

### HNSW index on s2_embedding

The migration adds an HNSW index on `s2_embedding` using cosine distance, matching the existing pattern for the `embedding` column. Set `statement_timeout = '300s'` during index creation. No changes needed to the existing `match_papers_by_embedding` RPC -- it uses the 1536-dim OpenAI embedding, not the 768-dim SPECTER embedding (different vector spaces).

### Enrichment DAG scheduling

The DAG is **scheduled daily** (`schedule='0 10 * * *'`), so that newly ingested papers are enriched on subsequent runs. Papers needing enrichment are identified by `s2_ids IS NULL`. Papers not found by S2 get `s2_ids = {}`, which excludes them from the query. No retry mechanism for not-found papers.

### S2 `externalIds` key mapping

The S2 API returns `externalIds` with PascalCase keys: `DOI`, `ArXiv`, `PMID`, `PMCID`, `DBLP`, `MAG`, `ACL`, `CorpusId`. The `to_s2_ids_dict()` method must map these to lowercase JSONB keys: `doi`, `pmid`, `pmcid`, `dblp`, `mag`, `acl`. The `corpus_id` comes from the top-level `corpusId` field (not `externalIds.CorpusId`). The `arxiv_id` convenience field on the DTO is populated from `externalIds.ArXiv`.

### Batch response correlation

The S2 batch endpoint returns results in the same order as the input IDs. Null entries correspond to papers S2 could not find. The parser in `fetch_paper_metadata_batch` populates `arxiv_id` on each `S2PaperMetadata` DTO from `externalIds.ArXiv`. The caller (`enrich_papers_with_s2_data`) correlates DTOs back to input papers via `arxiv_id`, matching the pattern used in `fetch_paper_authors_batch` / `link_batch_authors`.

### pgvector List[float] assignment

SQLAlchemy + pgvector accepts a raw `List[float]` when assigning to a `Vector(768)` column. No explicit conversion needed in the enrichment DAG.

### database_session() duplication

The `database_session()` context manager is duplicated across DAG files. The new enrichment DAG will duplicate it again. This is a known pattern; consolidating into `shared.db` is out of scope.

### worker/dags/daily_arxiv_ingest_dag.py (MODIFIED)

#### Changes to fetch_new_arxiv_papers

Extract three additional fields from each arXiv XML entry:
- `categories`: from `<category term="...">` elements
- `doi`: from `<arxiv:doi>` element
- `journal_ref`: from `<arxiv:journal_ref>` element

Return them in each paper dict alongside existing fields.

#### Changes to paper creation loop

After `create_paper()` returns, update the paper's `classification` column via direct SQLAlchemy query within the same session:

```python
classification_dict = {}
if paper.get('categories'):
    classification_dict['arxiv_categories'] = paper['categories']
if paper.get('doi'):
    classification_dict['arxiv_doi'] = paper['doi']
if paper.get('journal_ref'):
    classification_dict['arxiv_journal_ref'] = paper['journal_ref']

if classification_dict:
    session.query(PaperRecord).filter(
        PaperRecord.paper_uuid == paper_dto.paper_uuid
    ).update({"classification": classification_dict})
```
